Q4. Print output:

var x = 21;
        var girl = function () {
            console.log(x);
            var x = 20;
        };
        girl ();

o/p => undefined

  • First a table is created having two columns one memory allocation and other as code execution.
  • The Js file executes line by line from top to bottom by storing variables as undefined at first.
  • In Memory Allocation table x is stored as undefined, skips the function definition untill its called.
  • Then at last it comes to function named girl and make another table inside code execution having two columns one memory allocation and other as code execution.
  • Now in code execution we get x value as 21. so, in memory allocation x value is replaced to 21 in place of undefined.
  • Now again it comes at function girl and see console.log statement to print x , as X is called before declaring the variable with in the function . x returns as undefined(because of hoisting).

  • Next line is function b, in it as x value is not declared. x value get redeclared and becomes 40 . Giving o/p as 40

Q7. Write a function that accepts parameter n and returns factorial of n

        function factorial(n) {
            var fact = 1;
            for(i=n; i>0;i--){
                fact*=i;
            }
            return(fact);
        }
        console.log(factorial(5))
